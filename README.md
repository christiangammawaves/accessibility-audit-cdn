# accessibility-audit-cdn

Source code and CDN distribution for the WCAG 2.2 Level AA accessibility audit toolkit.

## What This Is

A browser-based accessibility audit system with 63 audit modules that runs inside Claude via Playwriter. Scripts are bundled into a single file and served from jsDelivr CDN, injected into web pages via `page.addScriptTag({ url })`. The system analyzes the DOM and accessibility tree, detecting issues with automatic false positive filtering and confidence scoring.

Built and maintained by the Gamma Waves Development Team.

## CDN URL

```
https://cdn.jsdelivr.net/gh/christiangammawaves/accessibility-audit-cdn@v13.2.0/dist/a11y-audit-bundle.min.js
```

## Repo Structure

| Path | What It Is |
|------|------------|
| `scripts/` | Core audit engine (5 files) — version, helpers, verifier, init, snapshot-analyzer |
| `components/` | 63 files (47 page components + audit modules + utilities) |
| `tests/` | 574 automated tests (Vitest) — unit, regression, and contract tests |
| `dist/` | Built bundle files (generated by `build-cdn-bundle.sh`) |
| `audit-bundle.js` | Component registry for detecting and running page component audits |
| `full-audit-orchestrator.js` | 8-phase orchestrated audit with snapshot-first workflow |
| `learned-exceptions.json` | False positive patterns (schema v1.1.0) |
| `build-cdn-bundle.sh` | Build script — concatenates source files into CDN bundle |
| `validate-bundle.js` | Bundle validation (13 checks — globals, version, component count) |

## Build & Deploy

After any source file change:

```bash
# 1. Rebuild the bundle
bash build-cdn-bundle.sh

# 2. Commit and tag
git add -A
git commit -m "build: update CDN bundle to vX.Y.Z"
git tag vX.Y.Z
git push origin main --tags

# 3. Verify jsDelivr serves it
curl -I "https://cdn.jsdelivr.net/gh/christiangammawaves/accessibility-audit-cdn@vX.Y.Z/dist/a11y-audit-bundle.min.js"
```

## Testing

```bash
npm install
npm test              # Run all 574 tests
npm run test:watch    # Watch mode
npm run test:coverage # Coverage report (v8)
```

## Version

Version source of truth: `scripts/version.js`

When bumping the version:
1. Update `version`, `releaseDate`, and `@version` tag in `scripts/version.js`
2. Update `package.json` version
3. Rebuild the bundle
4. Update the CDN URL version tag in the skill repo's `SKILL.md` if the major/minor version changed
